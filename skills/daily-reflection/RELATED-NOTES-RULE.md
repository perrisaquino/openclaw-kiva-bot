# RELATED NOTES RULE - Critical Implementation

## 🎯 **Core Principle:** 
**Connect new insights to existing knowledge FIRST. Stop making up note titles!**

## 📋 **Process for Every Reflection:**

### **1. VAULT SEARCH (Required First Step)**
Before writing "Related Notes" section, ALWAYS run these searches:

```bash
# Search for semantic connections to reflection content
find "/Users/iamperris/Local Documents/Main Second Brain (Local)/" -name "*.md" -type f -exec grep -l "KEYWORD1\|KEYWORD2\|KEYWORD3" {} \; | head -10
```

**Use keywords from the reflection:**
- Core concepts (e.g., "thriving artist", "mentoring", "CyphrCam")
- Names mentioned (e.g., "Anthony", "Franz", "Jeyna")  
- Themes (e.g., "ADHD", "Filipino", "decolonization", "business model")
- Emotional states (e.g., "frustrated", "excited", "aligned")

### **2. SECTION STRUCTURE (Required Format)**
```markdown
## 🔗 Related Notes

### **Existing Vault Connections:**
- [[Real Note Title]] - How it connects to this reflection
- [[Another Real Note]] - Why it's relevant

### **Potential New Notes:**  
- [[Future Note Idea]] - What this could become
- [[Another Potential Note]] - Topic to develop later
```

## ✅ **Quality Check:**
- **EXISTING section must come first** and contain REAL files found in vault search
- **Minimum 3-5 existing connections** before adding any "potential" notes
- **Check that linked files actually exist** - no made-up titles
- **Clear connection explanation** - why this existing note relates

## ❌ **Never Do This Again:**
- Making up note titles that sound plausible but don't exist
- Putting "potential notes" first  
- Skipping the vault search step
- Generic connections without specific relevance

## 🧠 **Why This Matters:**
The whole point of a second brain is **connecting new thoughts to existing knowledge**. That's how patterns emerge and insights compound. 

**Making up links = breaking the knowledge graph.**
**Real links = building authentic understanding.**
