## 🎯 QUESTION CATEGORIES

### **🛠️ DEVELOPMENT & BUILDING QUESTIONS**
*Use when daily note mentions coding, debugging, features, technical work*

#### Breakthrough Questions:
- "What breakthrough did you have with CyphrCam today that surprised you?"
- "What clicked for you technically today that had been confusing before?"
- "If you had to explain today's development challenge to a dancer, how would you put it?"
- "What did building today teach you about yourself as both artist and entrepreneur?"

#### Problem-Solving Questions:
- "What problem did you solve today that other solo builders would relate to?"
- "How did your approach to today's technical challenge reflect your dance training?"
- "What would you tell yourself from 6 months ago about the problem you faced today?"
- "If this development challenge was a movement pattern, how would you describe it?"

#### Learning Questions:
- "What did you learn about your users today from building this feature?"
- "How did today's work change your vision for what CyphrCam could become?"
- "What assumption about app development got challenged today?"

## 🤖 QUESTION SELECTION ALGORITHM

### **Daily Note Analysis:**
1. **Extract key themes** from content sections
2. **Identify emotional tone** (excited, frustrated, curious, etc.)
3. **Note specific challenges, breakthroughs, or insights**
4. **Check for mentions** of users, feedback, technical work, practice

### **Question Selection Rules:**
- **Maximum 5 questions** per day (don't overwhelm)
- **Mix categories** based on day's content (2-3 categories max)
- **Reference specific details** from the daily note
- **Vary question types** (don't repeat patterns daily)
- **Build on previous responses** when relevant

### **Example Selection Logic:**
```
IF daily_note_contains("debugging", "code", "feature"):
    SELECT 2 questions FROM development_questions
    
IF daily_note_contains("practice", "dance", "movement"):
    SELECT 2 questions FROM movement_questions
    
IF daily_note_contains("integration", "alignment", "split"):
    SELECT 1 question FROM integration_questions
    
ALWAYS include 1 question FROM story_questions FOR content_generation
```

### **Follow-up Logic:**

- **Build narrative arcs** (e.g., from frustration to breakthrough)
